DB접근 및 문제해결

DB에 접근하는 데에 초기 오류가 다수 발생하였고, 이를 해결하는 과정을 통해 현실 프로젝트 수행능력을 길렀습니다.

1. DB Security Group Inbound IP 포트 3306 개방

2. DB root 유저 host 비제한 추가 
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'password' WITH GRANT OPTION;

3. DB host=='%'인 root유저 비밀번호 변경 
UPDATE mysql.user SET password=PASSWORD("NEWPASSWORD") WHERE user='root' AND host='%';
FLUSH PRIVILEGES;
4. MySQL 서비스 재시작 
sudo service mysql restart
5. my.cnf 파일 수정 
path : /etc/mysql/my.cnf
comment out "bind-address = 127.0.0.1"
6. MySQL Workbench 이용 접속

데이터베이스 실습

기존 데이터베이스 수업 실습과제의 프론트엔드 부재라는 한계점을 보완하기 위하여 실제 운영중인 웹사이트의 DB를 분석해보고, SQL문을 이용하여 데이터베이스 실습을 진행하였습니다.

1. 유저 이름 불러오기 
SELECT user_name FROM admin.xe_member;
2. 게시글 전체 확인 
SELECT * FROM admin.xe_documents;
3. 특정 게시글 조회수 변경 
UPDATE admin.xe_documents SET readed_count=897 WHERE document_srl=2220;
4. 특정 이름을 가진 회원이 작성한 총 댓글 수 
SELECT COUNT(user_name) FROM admin.xe_comments WHERE user_name='이규현';
5. 생일이 6월 7일인 회원이 작성한 게시글 
SELECT * FROM admin.xe_comments, admin.xe_member WHERE xe_member.birthday LIKE '____0607' and xe_member.member_srl=xe_comments.member_srl;
